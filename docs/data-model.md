# Data Model

## Overview

Honeygen persists structured metadata in SQLite and stores generated bytes on disk. The main runtime entities are:

- world models
- generation jobs
- assets
- events

## World models

Stored in `world_models`.

Key fields:

- `id`
- `name`
- `description`
- `json_blob`
- `created_at`
- `updated_at`

The JSON payload shape matches `backend/internal/worldmodels/types.go`.

Required payload sections:

- `organization`
  - `name`
  - `industry`
  - `size`
  - `region`
  - `domain_theme`
- `branding`
  - `tone`
  - `colors` is optional and normalized to `[]`
- `departments[]`
- `employees[]`
  - `name`
  - `role`
  - `department`
- `projects[]`
- `document_themes[]`

Seeded default:

- `id`: `northbridge-financial`
- source: `sample-data/world-models/northbridge-financial.json`

## Generation jobs

Stored in `generation_jobs`.

Key fields:

- `id`
- `world_model_id`
- `status` (`pending`, `running`, `completed`, `failed`)
- `started_at`
- `completed_at`
- `summary_json`
- `error_message`
- `created_at`
- `updated_at`

`summary_json` contains:

- `manifest_count`
- `asset_count`
- `categories[]`
- `logs[]`

Each log entry includes:

- `time`
- `level`
- `message`
- optional `path`
- optional `category`

## Assets

Stored in `assets`.

Key fields:

- `id`
- `generation_job_id`
- `world_model_id`
- `source_type`
- `rendered_type`
- `path`
- `mime_type`
- `size_bytes`
- `tags_json`
- `previewable`
- `checksum`
- `created_at`

Notes:

- `path` is stored relative to the storage root, usually under `generated/<world-model-id>/<job-id>/...`
- `previewable` is `true` only for safe inline formats
- `rendered_type` currently includes `html`, `markdown`, `text`, `csv`, and `pdf`

## Events

Stored in `events`.

Key fields:

- `id`
- `asset_id`
- `world_model_id`
- `event_type`
- `method`
- `query`
- `path`
- `source_ip`
- `user_agent`
- `referer`
- `status_code`
- `bytes_sent`
- `timestamp`
- `level`
- `metadata_json`
- `created_at`

Events are written by the API when `decoy-web` forwards request telemetry to `/internal/events`.

Typical event characteristics:

- `event_type`: `http_request`
- `path`: `/generated/<world-model-id>/<job-id>/...`
- `status_code`: HTTP status returned by the decoy service

## Storage layout

### SQLite

- default path: `/app/storage/sqlite/honeygen.db`
- Docker volume: `sqlite-data`

### Generated files

- API and decoy container path: `/app/storage/generated`
- admin download mount path: `/usr/share/nginx/html/generated`
- Docker volume: `generated-assets`

Example generated file path:

```text
generated/northbridge-financial/<job-id>/public/about.html
```

Direct decoy URL for that file:

```text
/generated/northbridge-financial/<job-id>/public/about.html
```

## Relationship summary

- one world model -> many generation jobs
- one generation job -> many assets
- one asset -> zero or many events
- one world model -> many assets and events through those generated outputs
